# Synthetic Behavioral Evidence

## Status

The recorded development matrix exercised eleven pressure, ownership, failure-state, standalone-policy, and trigger cases. The strongest supported conclusion is that Code Territory Guide preserved or improved the rubric-scored behavior without a recorded regression. Earlier prose-only runs could not prove repository-changing behavior, which is why the canonical harness now materializes isolated writable Git fixtures.

This is historical development evidence. The current artifact, delivery, multi-repository, and commit-convention policies require fresh canonical baseline/treatment runs before claiming behavioral coverage for those capabilities.

## Qualification

The historical matrix was adaptive skill development, not a frozen causal experiment. Some early runs used prose fixtures or unsupported routing and some judge attempts failed before valid reruns were captured. Those artifacts remain local rather than being presented as release evidence.

Future reports should be regenerated from fresh materialized baseline and installed-skill runs with:

- a frozen treatment tree hash
- identical fixture and routing snapshots across paired arms
- observable worktree diffs and validation commands
- preserved exclusions and failed attempts
- independent criterion-level judgments

Run `python evals/build_report.py` after producing fresh valid judgments to replace this summary with current counts.
